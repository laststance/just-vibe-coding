# PRD: vibe coding ✨

## 1. 概要（Why）

**vibe coding** は、
> 目的も成果もなく、ただコードを書くための場所
を提供する VS Code 拡張です。

ソフトウェア開発（設計・仕様・レビュー・影響範囲）と切り離し、
**「喋るみたいにコードを書く」** 体験を作ります。

- 🧠 考える前に手が動く
- 🫧 失敗・破壊・未完成が許される
- 🎧 console が相づちを打つ

---

## 2. 問題定義（Problem）

### 現状の課題

- コードを書くこと自体は好き
- しかし実際には：
  - 既存プロジェクト → 文脈理解が重い
  - 新規プロジェクト → アイデア・設計・仕様が必要
- 結果として **「ただコードを書く」入口がない**

### 本質的な問題

> **コードを書くこと** と **ソフトウェア開発** が強く結びつきすぎている

これは、
- 喋る = 演説 / 漫才
が前提になっていて、
**雑談する場所がない** 状態に近い。

---

## 3. ゴール（Goals）🎯

### プロダクトゴール

- ワンアクションで「コードを書く空間」に入れる
- 開いた瞬間に、もう書ける
- 書いたらすぐ反応が返る

### UXゴール

- 目的を聞かれない
- 名前をつけさせられない
- 意味を与えられない

---

## 4. 非ゴール（Non-Goals）❌

vibe coding は以下を**提供しない**：

- プロジェクト管理
- フレームワーク選択
- テンプレート選択
- README / ドキュメント生成
- Lint / Format / Test
- 生産性向上KPI

👉 **vibe を壊すため**。

---

## 5. ターゲットユーザー 👤

### メインユーザー

- コードを書くのが好きなエンジニア
- 大きなコードベースに疲れている人
- アイデアがなくても手を動かしたい人

### 利用シーン

- 仕事前・仕事後のウォームアップ
- 思考がまとまらない時
- 何も作る気が起きないけどコードは書きたい時

---

## 6. ユースケース（User Stories）📖

### US-1: なんとなくコードを書きたい

> コマンド一発で、
> 誰にも影響しない空間で、
> console.log を打ちたい

### US-2: 既存プロジェクトから逃げたい

> 今開いている巨大なリポジトリを忘れて、
> 別世界でコードを書きたい

### US-3: 保存や成果を気にしたくない

> 消してもいい、
> 読み返さなくてもいい、
> その瞬間だけで完結したい

---

## 7. 機能要件（Functional Requirements）⚙️

### FR-1: セッション開始

- Command Palette に以下を提供
  - `Vibe Coding: Start`

### FR-2: 専用ワークスペースへの切り替え

- 実体ディレクトリ例：
  ```txt
  ~/.vibe-coding/
    └─ sessions/
        └─ 2026-01-23-17-42/
            └─ vibe.ts
  ```

- 現在開いているプロジェクトとは**物理的に分離**

### FR-3: 単一ファイルの提供

- デフォルトファイル：`vibe.ts`（または `vibe.js`）
- 初期内容：
  ```ts
  console.log("vibe coding")
  ```

### FR-4: 即時実行 & Console表示

- 保存 or 実行で即 Node.js 実行
- Output / Terminal に結果を表示

---

## 8. 非機能要件（Non-Functional Requirements）🧘

- 🫧 心理的安全性が最優先
- ⚡ 起動は高速（1秒以内）
- 🔕 通知・警告は最小限
- 🧼 クリーンアップ可能（後述）

---

## 9. UX / UI 方針 🎨

### 原則

- 意味を与えない
- 選択させない
- 判断を迫らない

### UI要素

- エディタ + Console の2ペイン
- ステータスバー表示：
  - `~ vibe coding ~`

---

## 10. MVPスコープ 🚀

### v0.1（必須）

- [x] コマンド `Vibe Coding: Start`
- [x] 専用ディレクトリ生成 & 切り替え
- [x] 単一ファイル生成
- [x] Node.js 実行
- [x] Console 出力

### やらないこと

- 設定画面
- 言語選択
- セッション管理UI

## 11. 将来拡張（Future Ideas）🔮

## 12. 成功指標（Success Metrics）📈

定量KPIは持たない。

代わりに：

- 「今日はちょっとコード書けたな」と思える
- IDEを閉じる時に疲労が増えていない

それで十分。

---
